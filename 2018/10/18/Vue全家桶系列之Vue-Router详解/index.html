Veröffentlicht am 2018-10-18 | in Vue | Visitors:

介绍

使用 Vue 的同学对 Vue-Router 肯定不会陌生。这是一个 Vue 官方的路由管理器。包含的功能有

  • 嵌套的路由/视图表
  • 模块化的、基于组件的路由配置
  • 路由参数、查询、通配符
  • 基于 Vue.js 过渡系统的视图过渡效果
  • 细粒度的导航控制
  • 带有自动激活的 CSS class 的链接
  • HTML5 历史模式或 hash 模式，在 IE9 中自动降级
  • 自定义的滚动条行为

Vue-Router 能帮我们更好通过url地址（路由）的控制页面组件的加载。官网其实已经写的非常简单易聊，这里部分转自官网加上一些自己的理解，帮助自己和其他人理解。官方中文文档传送门。同时推荐大家浏览器安装尤大写的Vue-DevTools，用了才知道有多棒。

动态路由匹配

动态路由匹配的意思其实就是在路由上增加参数用来把同一个组件渲染不同的数据。比如用户信息的列表，分页等等。

在组件中访问的使用 this.$route.params.id 来访问id，记得是 $route 不是 $router。
还有一种一种是url请求中的 query，比如我们现在访问的路由是 localhost:8080/user/3?gender=male&name=slash，这里要拿到信息 3 是id 通过 $route.params ?后面的部分则是通过 $route.query。返回的都是一个对象

模式 匹配路径 $route.params $route.query
/user/:username /user/evan { username: ‘evan’ } undefined
/user/:username/post/:post_id?gender=male&age=29 /user/evan/post/123 { username: ‘evan’, post_id: 123 } {gender: ‘male’, age: 29}

需要注意的是如果是同一组件被复用，两个路由都渲染同个组件，比起销毁再创建，复用则显得更加高效。不过，这也意味着组件的生命周期钩子不会再被调用。如果需要对路由参数的变化做出相应的话，可以使用 watch 监测 $route 对象。

编程式导航

除了使用 <router-link> 创建 a 标签来定义导航链接，我们还可以借助 router 的实例方法，通过编写代码来实现。

有时候我们需要在 Vue 组件中通过 JS 的方式去控制，这就可以用到前面说到的 this.$router 它是路由器的实例，我们可以通过 this.$router.push 或者 this.$router.replace 控制路由跳转。

this.$router.push(...) 使用等同于点击 <router-link :to="...">。

他们还支持第二和第三个参数 onComplete 和 onAbort，这是两个回调函数，会在导航成功完成 (在所有的异步钩子被解析之后) 或终止 (导航到相同的路由、或在当前导航完成之前导航到另一个不同的路由) 的时候进行相应的调用。

this.$router.push 和 this.$router.replace不同的地方在于， replace 不会向 history 添加新记录，而是跟它的方法名一样 —— 替换掉当前的 history 记录。

另外 $router还有一个 go() 方法。

这个方法的参数是一个整数，意思是在 history 记录中向前或者后退多少步。

重定向和别名

重定向也是通过 routes 配置来完成，比如有时候我们访问/home 以及根目录的时候都希望访问的是 首页 /home 路由。

const router = new VueRouter({
routes: [
{ path: ‘/‘, redirect: ‘/home’ }
]
})

此时访问 / 时 路由会改成 /home

别名是alias

别名不会改变路由 /b 的时候也是访问的 /a

HTML5 History 模式

vue-router 默认 hash 模式 —— 使用 URL 的 hash 来模拟一个完整的 URL，于是当 URL 改变时，页面不会重新加载。

如果不想要很丑的 hash，我们可以用路由的 history 模式，这种模式充分利用 history.pushState API 来完成 URL 跳转而无须重新加载页面。

当你使用 history 模式时，URL 就像正常的 url，例如 http://yoursite.com/user/id，也好看！
