webpack4.0简介

Diposting di 2018-07-18 | Di webpack | Pengunjung:

介绍

webpack 本质上是一个现代 javascript 应用程序的静态模块打包器(module bundler)。当 webpack 处理应用程序时，它会递归地构建一个依赖关系图(dependency graph)，其中包含应用程序需要的每个模块，然后将所有这些模块打包成一个或多个 bundle。

本文主要记录的是学习 webpack4.1x+ 官方文档的笔记。

从webpack4.0 开始，webpack 已经不需要引入一个配置文件，但 webpack 依然是高自由配置的。核心的概念有 4 个：

  • 入口（entry）
  • 输出（output）
  • loader（module.rules）
  • 插件（plugins）

概念

loader

loader 让 webpack 能够去处理那些非 Javascript 文件（webpack 自身只理解 JavaScript）。简单讲就是让 webpack 可以识别除了 .js 外的其他文件，比如 .css,.styl,.png,.svg 等等。loader 是写在 module 下的 rules，rules 接受一个对象数组，对象包含2个主要属性：

  • test 用于表示出应该被对应的 loader 进行转换的某个或者某些文件，一般使用正则
  • use 表示进行转换时，应该使用哪个 loader。可以是字符串或者数组。
