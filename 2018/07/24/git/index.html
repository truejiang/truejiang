简介

git是一个分布式的版本管理系统，用于管理Linux内核开发而创立。使用起来非常便捷，配合 Github 或者 码云 使用可以方便随时管理自己的文档版本。下面我们先介绍下用法。

安装git可以点击前面的链接，选择系统类型下载安装即可。

提交修改

在这里先讲下 git 的基本概念。Git 分为下面三个区：

  • 工作区(Working Directory)：我们可以直接看到的文件内容。
  • 暂存区(Stage)：一般存放在 .git 目录下的 index 文件，我们 add 的时候就是暂时放在这里
  • 版本库(Repository)：工作区根目录下.git隐藏目录。用于存放我们 commit 后的版本

根据上面的分区，我们先在该目录下新建一个文件夹名为 firstFile.txt。并在里面添加一些内容 => ‘my first git files’。然后添加以及提交这个文件。

  • git add 表示提交指定修改文件到暂存区， git add . 表示提交所有的修改。
  • git commit -m 将当期提交到暂存区的所有内容，一起提交到版本库，-m 后面有空格， description 需要用引号括起来，并写好提交的原因和内容。

分支

分支的概念：分支是为了将修改记录的整体流程分叉保存。分叉后的分支不受其他分支的影响，所以在同一个数据库里可以同时进行多个修改。
我们知道，如果同时多人开发时，我们想要 push 某一个版本上去，但同时可能还有其他正在开发另外一个版本，这样容易冲突影响其他开发人员。
这时我们就可以创建一个分支，作为自己专用。也可以用作一个项目的几个版本保存，比如 开发版本，bug版本，稳定版本等，都可以单独作为分支。
