AJAX的API和实现的原理.md

Diposting di 2018-06-27 | Di JavaScript | Pengunjung:

含义

Ajax 即“Asynchronous Javascript And XML”（异步 JavaScript 和 XML），是指一种创建交互式网页应用的网页开发技术。主要作用是在不重新加载网页的情况下，对网页某部分进行更新，提高用户体验。

XMLHttpRequest 对象

ajax请求通信的核心是 XMLHttpRequest 对象，一个简易的请求分为四步

  • 第一步：创建一个 XMLHttpRequest 对象的实例， let xhr = new XMLHttpRequest()
  • 第二步：调用实例对象的 open() 方法来初始化一个请求，该方法接收3个参数，另外还可以接收第四和第五个参数，别分是用户名和密码但没有使用过，具体可以看MDN文档 ：
    • method：请求使用的HTTP方法，如 ‘GET’,’POST’ 等等，也可以小写
    • url：该次请求要访问的URL地址
    • async：是否进行异步操作，接收的是一个布尔值
  • 第三步：调用实例对象的 send() 方法发送请求，如果 open() 的第三个参数为 true，表示异步请求，则该方法会理解返回。否则，是同步请求，会直到请求的响应完全接受以后，该方法才返回。该方法也接受一个参数，参数为请求发送的数据，一般是POST请求时需要发送，如果不需要发送数据，则必须传入 null，调用该方法后，会立即向服务器派发请求
  • 第四步：在请求发送后，响应的数据就会自动填充 XMLHttpRequest 实例对象的属性。其中有用的以下几个：
    • responseText：请求返回的文本
    • responseXML：如果响应的内容类型是 ‘text/xml’ 或者 ‘application/xml’，这个属性将保存包含着响应数据的 XML DOM 文档
    • status：响应的 HTTP 状态，一般状态码是 200 表示成功，304 则代码请求的资源没有被修改，可以使用浏览器缓存的版本，也表示响应是有效的
    • statusText：HTTP状态的说明

  • 0：未初始化，即还没有调用 open() 方法
  • 1：启动，已经调用 open() 方法，但还未调用 send() 方法
  • 2：发送，已经调用 open() 和 send() 方法，但尚未接收到响应
  • 3：接收，已经接收到部分响应数据
  • 4：完成，已经接收到全部的响数据

GET 请求

GET 请求是一种常用的请求类型，用于向服务器查询某些信息。一般将查询字符串参数追加到 URL 的末尾，格式为 url?queryName1=queryValue1&queryName2=queryValue2 (?表示查询字符串和url的衔接处，后面每组查询参数的键和值之间用 = 连接，每组查询参数之间用 & 连接，最后一组不需要加 &)，需要注意的是当通过 open() 方法传入 URL末尾的查询字符串必须经过 encodeURIComponent() 进行正确的编码才行。

进度事件

进度事件包含了很多个，我们主要看下 load 和 progress 事件

  • load事件：表示响应完成，相当于简化了 onreadystatechange 中监听 readyStatus === 4 这个步骤
  • progress事件：会根据响应的进度去周期性的触发，利用这个特性我们可以写一个响应的进度条。

load 事件不多写代码了，实际就相当于上面说到的简化代码的一个语法糖。

progress 事件会接收一个 event 对象。其中有用的属性有:

  • lengthComputable：表示进度信息是否可以用的布尔值，即下面2个属性是否可用
  • loaded：当前已接收的响应数据字节
  • total：根据 Content-Length 响应头部确定的预期字节数

根据上面的几个属性，我们可以通过监听去写一个加载数据的进度条，代码这里就不写了，大家可以自己尝试些下，原理就是根据 loaded 的值除以 total 能得到当前数据响应进度的百分比，再去给进度条上显示对应的百分比长度。
另外需要提醒的是 onprogress 必须放在 open() 和 send() 前面，而且 open() 的 async 参数要设置成 true
